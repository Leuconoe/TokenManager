# TokenManager — 토큰 기록 관리 앱 (Plan)

> Context Anchor
> - **WHY**: API키/토큰 탈취 공급망 공격 증가 → 발급한 토큰의 "수명"을 사람이 추적 못 해 방치되는 게 근본 원인. 만료 추적 + 위생(hygiene) 경고로 노출 표면을 줄인다.
> - **WHO**: 여러 서비스(GitHub, AWS, 클라우드, CI 등)에 토큰을 발급하는 개발자/운영자 본인.
> - **RISK**: 앱 자체가 토큰 저장소가 되면 탈취 표적이 됨 → **토큰 값은 절대 저장 안 함(metadata-only)**.
> - **SUCCESS**: 만료 임박/만료/무기한 토큰을 놓치지 않고 알림받아 회전(rotate)·폐기한다.
> - **SCOPE**: 로컬 전용, 오프라인, 자동연동 없음. 메타데이터 CRUD + 암호화 + 알림.

## 1. 핵심 설계 원칙

| 원칙 | 내용 |
|------|------|
| **Metadata-only** | 토큰 값 자체는 저장 안 함. 저장: 서비스명/만료일/노트뿐. |
| **No auto-sync** | OAuth/API 자동연동 배제 (사용자 결정 존중). 자동연동 시 앱이 토큰을 보유 → 그 자체가 탈취 표적. |
| **암호화 at-rest** | DB 전체 암호화. 키는 Android Keystore(하드웨어 TEE/StrongBox) 보관 — 칩 밖으로 안 나감. **코드에 시드/키 일절 없음** → 디컴파일해도 복원 불가. |
| **No hardcoded seed** | 키는 최초 실행 시 기기에서 랜덤 생성·하드웨어 격리. APK·소스 어디에도 키 재료 없음. 기기 하드웨어 종속이라 타 기기 복사 시 복호화 불가. |
| **생체인증 게이트** | 앱 진입 시 BiometricPrompt(지문/얼굴) 또는 기기 PIN으로 잠금 해제 후에만 복호화. |
| **토큰 입력 경고(비차단)** | 노트에 토큰 형태 문자열 감지 시 **경고만** 표시. 저장은 사용자 선택 존중(차단 안 함). |

### ✅ "코드 시드값이 아닌 기기 고유값으로, 디컴파일해도 복원 불가" — Keystore로 충족
사용자 핵심 요구: **(a) 코드에 박힌 시드값이면 안 됨, (b) 기기 고유값 기반, (c) 디컴파일 시 복원 불가.** → Android Keystore가 셋 다 만족하며, 읽을 수 있는 식별자(MAC/ANDROID_ID)보다 강함.

| 방식 | 코드 시드 없음 | 기기 종속 | 디컴파일 방어 | 비고 |
|------|:---:|:---:|:---:|------|
| 코드 하드코딩 키 | ❌ | ❌ | ❌ | APK에서 추출됨 — 최악 |
| MAC/ANDROID_ID 파생 | ✅ | △ | ❌ | **읽을 수 있는 값** → 데이터 탈취자도 동일하게 읽음. Android 6+ MAC은 더미 고정값이라 사실상 무의미 |
| **Android Keystore (채택)** | ✅ | ✅(하드웨어) | ✅ | 키가 TEE/StrongBox 밖으로 안 나감. 읽기 불가, 칩 내부 연산만. 디컴파일·adb 추출·타기기 복사 모두 무력화 |

- 키는 최초 실행 시 **기기에서 랜덤 생성** → flutter_secure_storage(Keystore 래핑)에 저장. 소스/APK 어디에도 키 재료 없음.
- `setUserAuthenticationRequired` 적용 시 **하드웨어 + 생체인증** 동시 충족해야 키 동작.

## 2. 기술 스택 (Flutter)

| 영역 | 선택 | 비고 |
|------|------|------|
| UI | Flutter + Material 3 | 단일 코드베이스, 향후 iOS 확장 여지 |
| 상태관리 | Riverpod | 테스트 용이, 의존성 주입 |
| 로컬 DB | **Drift + SQLCipher** (`sqlcipher_flutter_libs`) | SQLite 전체 암호화 |
| 암호화 키 | **flutter_secure_storage** | 내부적으로 Android Keystore + EncryptedSharedPreferences 사용. DB 암호키(랜덤 256bit) 자체를 안전 저장 |
| 생체인증 | **local_auth** | BiometricPrompt + 기기 자격증명 fallback |
| 알림 | **flutter_local_notifications** + **workmanager** | 만료/무기한 주기 점검 → 로컬 알림 (서버 푸시 아님) |
| 토큰 패턴 탐지 | 자체 정규식 + 엔트로피 검사 | 노트 저장 전 검증 |

> **푸시 알림 방향 정정**: "푸시"는 서버(FCM)가 아니라 **로컬 스케줄 알림**으로 구현. 서버가 없으니(=로컬 전용) FCM 불필요하고, 메타데이터를 외부로 안 보내는 게 보안상 이점.

## 3. 데이터 모델

```
TokenEntry
├─ id            : String (UUID)
├─ serviceName   : String          (필수) 예: "GitHub PAT - CI 배포용"
├─ issuedAt      : DateTime?        (선택) 발급일
├─ expiresAt     : DateTime?        (선택) 만료일 — null = 무기한(경고 대상)
├─ note          : String          토큰 값 금지. 패턴 검증 통과해야 저장
├─ createdAt     : DateTime
└─ updatedAt     : DateTime
```

- `expiresAt == null` → "무기한" 상태로 분류, 주기적 보안 경고 대상.
- 상태 계산(파생): `valid` / `expiringSoon`(≤14일) / `expired` / `noExpiry`.

## 4. 기능 목록

### F1. 토큰 항목 CRUD
- 추가/수정/삭제/목록. 목록은 만료 임박순 정렬.
- 입력 필드: 서비스명, 발급일(선택), 만료일(선택), 노트.

### F2. 노트 토큰-입력 경고 (비차단)
저장 시 노트 검사 → 토큰 의심 패턴이면 **경고 표시(저장은 허용)**. 입력 여부는 사용자 선택:
- 알려진 prefix: `ghp_`, `github_pat_`, `gho_`, `sk-`, `xoxb-`, `AKIA`, `AIza`, `eyJ`(JWT) 등
- 고엔트로피 연속 문자열(예: 32자 이상 base64/hex 덩어리)
- 감지 시: "토큰 값으로 보이는 내용이 있습니다. 계속 저장할까요?" (저장 / 취소)
- 입력란 상단 고정 안내: **"⚠️ 토큰 값은 입력하지 마세요. 이 앱은 토큰 추적용이며 값 저장을 권장하지 않습니다."**

### F3. 암호화 저장 (2-키 모델)
일상 저장과 백업은 **서로 다른 키 체계**를 쓴다 — 보안과 연속성을 동시에 만족:

| 용도 | 키 출처 | 특징 |
|------|---------|------|
| **일상 at-rest** (F3) | Keystore 하드웨어 키 (기기 종속) | 비밀번호 불필요(생체인증만). 디컴파일/타기기 복원 불가 |
| **백업/복원** (F7) | 사용자 패스프레이즈 (PBKDF/Argon2id) | 기기 바뀌어도 비밀번호로 연속성 유지 |

- 최초 실행: 랜덤 256bit DB 키 생성 → flutter_secure_storage(Keystore) 저장.
- 이후: 생체인증 통과 → secure_storage에서 키 로드 → SQLCipher DB 복호화.
- Keystore 키는 기기 종속이라 백업에 쓸 수 없음 → 그래서 F7은 별도 패스프레이즈 키 사용.

### F4. 생체인증 게이트
- 앱 진입/백그라운드 복귀 시 잠금. local_auth 인증 성공 전 데이터 표시 안 함.
- 생체 미등록 기기: 기기 PIN/패턴 fallback.

### F5. 만료 알림
- workmanager 일 1회 점검 → flutter_local_notifications:
  - `expired`: "만료됨 — 지금 폐기/회전하세요"
  - `expiringSoon` (D-14, D-7, D-1): 사전 경고
- 알림 탭 → 해당 항목 상세로 이동.

### F6. 무기한 토큰 보안 경고
- `expiresAt == null` 항목에 대해 주기적(예: 주 1회) 경고 알림.
- 상세 화면에 "만료일 미설정 — 회전 정책 권장" 배지.

### F7. 백업 / 복원 (지원 확정)
- **암호화 export**: 전체 메타데이터를 **사용자 지정 패스프레이즈**로 암호화한 단일 파일로 내보냄.
  - 키 유도: **Argon2id** (또는 PBKDF2-HMAC-SHA256 高 iteration) + 랜덤 salt → AES-256-GCM.
  - ⚠️ Keystore 키는 기기 종속·추출 불가라 백업에 못 씀 → **이식성을 위해 패스프레이즈 기반 별도 암호화** 필요.
- **import/복원**: 파일 선택 → 패스프레이즈 입력 → 복호화 → 기존 DB에 병합/덮어쓰기 선택.
- export 파일 헤더에 버전·KDF 파라미터·salt·nonce 포함(평문), 본문은 암호문.
- 메타데이터만 들었으므로(토큰 값 없음) 백업 위험도 자체가 낮음 — 그래도 패스프레이즈 강제.

## 5. 보안 위협 모델 (요약)

| 위협 | 대응 |
|------|------|
| 기기 분실/탈취 | DB 암호화 + 생체인증 게이트. 잠금 해제 없이는 메타데이터 노출 안 됨 |
| 앱 데이터 추출(adb backup 등) | SQLCipher 암호문만 추출됨. 키는 Keystore(추출 불가) |
| **APK 디컴파일** | 코드에 시드/키 재료 0. 키는 실행 시 기기 생성·하드웨어 격리 → 정적 분석으로 복원 불가 |
| 루팅 기기 | StrongBox/TEE 키는 여전히 보호. + 루팅 탐지 경고(선택) |
| 사용자가 토큰을 노트에 저장 | F2 능동 차단 |
| 메타데이터조차 민감(어느 서비스에 토큰 있는지) | 로컬 전용·외부 전송 0. 자동연동 배제 |
